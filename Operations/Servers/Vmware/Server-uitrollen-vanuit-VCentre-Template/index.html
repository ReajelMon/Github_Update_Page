    Server uitrollen vanuit VCentre Template

    • Log-in VCentre

    • Klik op VM's en Templates

    • Zoek de template die je nodig hebt voor server 2016 of 2019 en klik met je rechter muisknop op New VM from this Template.
      Gebruik de template met Server 2019 in de naam.

    • Vul de naam in van de server die je gaat uitrollen MP-XXXXX en kik op next

    • Kies de gewenste ESX en klik op next

    • Kies de gewenste Datastore en klik op next (Medio eind 2020 gebruik de 3TB-datastores EIN03, EIN04, EIN05 en EIN07 bij voorkeur niet).

    • Vink Customize the operating system aan.

    • Kies de juiste Guest OS template en klik op next

    • Vul de juiste IP gegevens in die je in de IP lijst hebt bepaald en klik op next

    • Klik in het volgende scherm op Finish

    • De server wordt aangemaakt.

    • Wanneer Vcentre klaar is kun je de server aanzetten.

    • Nadat de server opgestart is wacht je een aantal minuten de server zal uit zichzelf herstarten, Vcentre zal de server automatisch registreren in het domein en de juiste licentie code toewijzen aan de server.
      Daarnaast wordt het IP adres ingesteld zoals je voorheen hebt opgegeven.

    (Mocht dit niet goed werken dan dient alles handmatig ingesteld te worden.
    Check dan ook nog even of adapter type op VMXNET3 staat. Zie ook het default wachtwoord voor Administrator in Passwordstate "Server local Admin" bij de notes.)

    • Hernoem de netwerknaam naar LAN:

    • Upgrade VMWaretools (soms wordt er om een reboot gevraagd, in sommige error-situatie komt VMWare met het voorstel om de VMWare-CD te mounten en kun je de installatie vanaf CD uitvoeren).

    • Zet "Auto Update" aan voor VMWare Tools voor de nieuwe VM.
      Ga in de Vsphere client de nieuwe server , tabblad updates en check of "Automatically upgrade on reboot" aan staat.

    • Rond de installatie als volgt af :

    • C-schijf vergroten naar 100GB.

    • Maak een E-schijf aan om de applicatie op te installeren

    • Maak eventueel benodigde extra partities aan en kies hierbij voor Partition structuur voor GPT.

    • Wissel de DNS-servers om als de server in Rotterdam staat.

    • Remote Desktop aanzetten.

    • Server in de in de juiste OU in AD zetten (Standaard NLDC).

    • Groep LocalAdmin_MP-xxxxx maken in AD maken en hier rs.PRTG in zetten. Deze groep lid maken van de lokale admin-groep van de nieuwe server

    • Groep LocalRDS_MP-xxxxx maken in AD maken. Deze groep lid maken van de lokale groep "Remote Desktop Users" van de nieuwe server.

    • SentinelOne installeren. \\10.10.202.60\software\SentinelOne )

    • RMM (\\10.10.202.60\software\RMM\Agent-Server ) installeren en Standaard Server Template toepassen.
      (Als de RMM-client reeds in de template zit dan alleen in het dashboard de naam wijzigen bijv. de MP-WEB-XXX).
      Draai binnen RMM de taak "ResetAdministratorPassword(servers) handmatig.

    • Installeer sysmon via "\mp-fs102.montapacking.nl\software\Sysmon\Sysmon-install\1 Install sysmon.cmd"

    • Installeer NXlog via het juiste script in "\mp-fs102.montapacking.nl\software\NXLog\Server Installatie". Het kan nodig zijn er een script bij te maken. Zie de lijst met configuraties in "\mp-fs102.montapacking.nl\software\NXLog"

    • Controleer of 7Zip en Notepad ++ zijn geïnstalleerd.

    • Remote Scheduled Tasks Management aanzetten in de firewall.

    • Server in PRTG toevoegen (kloon een soortgelijke server) (ook bij node PAP16-PRTG01, dan alleen de server met een ping-sensor)). Zie voor de interval-instellingen van de sensors (om onnodige belasting te voorkomen : "...\Handleidingen\Servers\Bijlagen\PRTG\PRTG Sensoren servers.xlsx".
      Als er een probleem optreedt met het toevoegen van WMI-sensoren dan even de "PRTG Probe Service" op de actieve PRTG-server (meestal de MP-PRTG01) onder een domainadmin draaien totdat de sensoren goed staan.
      Daarna de "PRTG Probe Service" weer onder "rs.prtg" laten draaien.

    • Server in juiste backup-pool toevoegen. Bij een SQL-server moet rs.veeam als login worden aangemaakt.

    • Windows updates uitvoeren.

    • Server (laten) toevoegen in lijst Remote Desktop Manager

    • Alleen de volgende cipher suites instellen bij een webserver die van buiten af bereikbaar is:
      |   TLSv1.2:
      |     ciphers:
      |       TLS_DHE_RSA_WITH_AES_128_GCM_SHA256 - strong
      |       TLS_DHE_RSA_WITH_AES_256_GCM_SHA384 - strong
      |       TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256 - strong
      |       TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384 -- strong

    Bij een server die alleen van binnen Monta bereikbaar is kan gewoon de best practice van IIS Crypto voor cipher suites geaccepteerd worden.

    Dit kan uitgevoerd worden middels IISCrypto allereerst op het tabblad "Schannel":

    • Vervolgens op het tabblad Cipher Suites (alleen in het geval van een server die van buitenaf bereikbaar is):

    \\10.10.202.60\data\8.0 Automatisering\1.0 Software\8.0.0 Diversen\IISCrypto.exe

    Voeg NXlog en sysmon toe voor de logging naar de SIEM bij Hunt & Hackett.

    Voor NXlog zie "\mp-fs102.montapacking.nl\software\nxlog".
    Installeer "nxlog-ce-3.1.2319.msi" en vervang in het bestand "c:\program files\nxlog\conf\nxlog.conf" de inhoud door de inhoud van betreffende configuratiefile uit "\mp-fs102.montapacking.nl\software\nxlog".
    Voor een standaard server zal dat de inhoud van "nxlog-sysmon.conf" zijn.

    Voor Sysmon zie "\mp-fs102.montapacking.nl\software\Sysmon\Sysmon-install". Voer "Install sysmon.cmd" uit.
