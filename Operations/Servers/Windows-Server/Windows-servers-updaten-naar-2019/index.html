    Windows servers updaten naar 2019

    • Log in op de Windows server welke ge-upgrade dient te worden naar Windows Server 2019;

    • Open een opdrachtprompt, ga naar c:\Windows\system32 en typ systeminfo.exe;

    Kopieer, plak en bewaar de resulterende systeeminformatie ergens buiten het apparaat;

    • Typ ipconfig / all in de opdrachtprompt en kopieer en plak de resulterende configuratiegegevens op dezelfde locatie als hierboven;

    • Open de Register-editor, ga naar de HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\WindowsNT\CurrentVersion-component en kopieer en plak de Windows Server BuildLabEx (versie) en EditionID (editie) op dezelfde locatie als hierboven.

    • Deinstalleer SentinelOne vanuit Dashboard SentinelOne

    • Nadat al de Windows Server-gerelateerde informatie is verzameld, maak je via vSphere een snapshot van de desbetreffende machine;

    • Verwijder via "Device Manager" de display driver;

    • Koppel via vSphere de installatie CD van Windows Server 2019 aan de machine welke te vinden is in de EIN-DATASTORE-ISO en ROT-DATASTORE-ISO;

    • Zoek de installatiemedia van Windows Server 2019 in "Deze Computer" en selecteer "setup.exe";

    • Selecteer "Ja" om het installatieproces te starten;

    • Selecteer voor apparaten met internetverbinding de optie "Updates, stuurprogramma's en optionele functies downloaden (aanbevolen)" en klik vervolgens op "Volgende";

    -

    • Selecteer de Windows Server 2019-editie die geïnstalleerd dient te worden en dan altijd (Desktop Experience) zodat de GUI geïnstalleerd wordt.
      Selecteer vervolgens "Volgende".

    • Selecteer "Accepteren" om de voorwaarden van de licentieovereenkomst te accepteren;

    • Selecteer "Persoonlijke bestanden en apps behouden" om ervoor te kiezen een in-place upgrade uit te voeren en selecteer vervolgens "Volgende";

    • Nadat "Setup" het apparaat heeft geanalyseerd, wordt er gevraagd om door te gaan met de upgrade door "Installeren" te selecteren;

    • De in-place upgrade start en toont het scherm Upgrading Windows met de voortgang. Nadat de upgrade is voltooid, wordt de server opnieuw opgestart.
      Mocht de upgrade blijven hangen dan kan het snapshot terug gezet worden en dan kan het nogmaals geprobeerd worden waarbij Windows Defender gedeinstalleerd wordt. Zie de bijlage.

    • Nadat de upgrade is voltooid, moet Windows Server 2019 opnieuw geactiveerd worden. Dit kan uitgevoerd worden middels het invoeren van één van de Server 2019 license keys welke te vinden zijn in Password State;

    • Voer Windows Updates uit.

    • Pas vervolgens de huidige OS-versie bij van de desbetreffende server in "IP-lijst" tabblad
      "NLDC | MP-IP-Lijst" en in VMware onder "Edit settings van VM > VM Options > General. Dit kan alleen als de server uit staat.

    • Herinstalleer SentinelOne

    Bijlage Uninstall Windows Defender

    Check Windows Defender is running on Windows Server with PowerShell

    If you want to check the Windows Defender status with PowerShell, use the Get-Service cmdlet.

    PS C:\> Get-Service -Name WinDefend

    Status Name DisplayName

    ------ ---- -----------

    Running WinDefend Windows Defender Service

    Copy

    We can see that Windows Defender is running on Windows Server. In the next step, we will remove Windows Defender on Windows Server.

    Uninstall Windows Defender on Windows Server

    Run PowerShell as administrator. Make use of the Uninstall-WindowsFeature cmdlet.
